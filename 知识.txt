根据阿里云的HaaS UI 完整离线官方文档
 
前置通用说明
 
1. 框架基础：基于 Vue 2.6.12 开发，面向 AIoT 带屏设备的跨平台渐进式 JS 应用框架，一端开发多端运行
2. CSS 规范：全量支持下文 6 大核心 CSS 样式模块，符合 W3C 标准子集，非支持属性不生效
3. 语法规则： v-model  双向绑定， @事件名  绑定事件， slot  插槽分发内容，Vue 组件内通过  this.$falcon  访问全局对象，全局上下文可直接调用  $falcon 
4. 运行环境：支持 AliOS Things、Linux、Windows、MacOS 系统，适配树莓派全系列、全志 D1 等主流 IoT 硬件
 
 
 
第一部分：HaaS UI 完整 CSS 样式规范（共 6 大核心模块）
 
模块 1：通用基础样式（所有组件全量支持）
 
核心说明
 
全局通用基础属性，控制组件的显隐、层级、定位、溢出规则，所有组件均可直接使用，完全兼容 W3C 标准
 
支持的全量属性
 
属性名 支持值 说明 
display  flex  /  none  /  block  /  inline-block  组件显示模式，默认  block  
visibility  visible  /  hidden  组件显隐，隐藏时仍保留占位 
opacity 0~1 数字 组件透明度，0 为完全透明 
overflow  visible  /  hidden  /  scroll  /  auto  内容溢出规则，仅固定宽高容器生效 
z-index 整数 组件堆叠层级，数值越大越靠前 
position  static  /  relative  /  absolute  /  fixed  定位模式， fixed  相对于屏幕视口定位 
left / right / top / bottom px 单位 / 百分比 定位偏移量，配合  position  使用 
pointer-events  auto  /  none  控制组件是否响应点击事件 
 
独有约束
 
-  position: fixed  不受父容器影响，始终相对于设备屏幕定位
-  overflow: scroll  仅在  scroller  组件或固定宽高的块级容器中生效
 
 
 
模块 2：文本样式（text 组件核心，全组件通用）
 
核心说明
 
控制文本的显示效果，仅  text  组件原生支持，其他组件内的文本必须嵌套  text  标签才能生效，完全兼容 W3C 标准
 
支持的全量属性
 
属性名 支持值 说明 
color 十六进制 / rgb / rgba / 标准颜色名 文字颜色 
font-size px 单位 文字字号，不支持 em/rem 单位 
font-weight  normal  /  bold  / 400~900 数字 文字粗细 
font-family 系统支持的字体名 文字字体，默认内置 FreeType2 矢量字库 
line-height px 单位 文字行高 
text-align  left  /  center  /  right  文字水平对齐方式 
text-decoration  none  /  underline  /  line-through  文字装饰线 
text-overflow  ellipsis  文字溢出省略，需配合  overflow:hidden  +  white-space:nowrap  使用 
white-space  normal  /  nowrap  文字换行规则 
 
独有约束
 
- 仅支持 px 固定单位，不支持相对单位（em/rem）
- 非  text  组件无法直接渲染文本，必须嵌套  text  标签
 
 
 
模块 3：盒模型样式（布局核心，全组件通用）
 
核心说明
 
控制组件的尺寸、内外边距，是页面布局的基础，完全兼容 W3C 标准盒模型
 
支持的全量属性
 
属性名 支持值 说明 
width px 单位 / 百分比 组件宽度 
height px 单位 / 百分比 组件高度 
min-width / max-width px 单位 组件最小/最大宽度 
min-height / max-height px 单位 组件最小/最大高度 
margin px 单位 / 百分比 组件外边距，支持 1~4 个值简写 
margin-left / margin-right / margin-top / margin-bottom px 单位 / 百分比 单方向外边距 
padding px 单位 / 百分比 组件内边距，支持 1~4 个值简写 
padding-left / padding-right / padding-top / padding-bottom px 单位 / 百分比 单方向内边距 
box-sizing  content-box  /  border-box  盒模型模式，默认  content-box  
 
独有约束
 
-  image  组件必须设置固定 width 和 height，不支持百分比宽高
-  scroller  组件必须设置固定宽高，否则无法触发滚动效果
 
 
 
模块 4：Flex 布局样式（Layout/div 核心，全容器组件支持）
 
核心说明
 
基于 W3C Flex 标准的弹性布局，是 HaaS UI 推荐的布局方式，所有容器组件（div/Layout）全量支持
 
支持的全量属性
 
容器属性（父元素：div/Layout）
 
属性名 支持值 说明 
display  flex  开启弹性布局，必填 
flex-direction  row  /  column  /  row-reverse  /  column-reverse  主轴方向，默认  row  
justify-content  flex-start  /  center  /  flex-end  /  space-between  /  space-around  主轴对齐方式 
align-items  flex-start  /  center  /  flex-end  /  stretch  /  baseline  交叉轴对齐方式 
flex-wrap  nowrap  /  wrap  /  wrap-reverse  换行规则，默认  nowrap  
align-content  flex-start  /  center  /  flex-end  /  space-between  /  space-around  /  stretch  多根轴线的对齐方式 
 
子项属性（子元素：div/LayoutItem）
 
属性名 支持值 说明 
flex 数字 子项占比权重，同  flex-grow  
flex-grow 数字 子项放大比例，默认 0 
flex-shrink 数字 子项缩小比例，默认 1 
flex-basis px 单位 子项在主轴上的初始大小 
align-self  auto  /  flex-start  /  center  /  flex-end  /  stretch  /  baseline  单独设置当前子项的交叉轴对齐方式 
 
独有约束
 
-  Layout  组件的直接子元素必须是 LayoutItem，默认  flex:1  等分布局
- 不支持 Grid 网格布局，仅支持 Flex 弹性布局
 
 
 
模块 5：背景与边框样式（全组件通用）
 
核心说明
 
控制组件的背景、边框、圆角效果，完全兼容 W3C 标准子集
 
支持的全量属性
 
属性名 支持值 说明 
background-color 十六进制 / rgb / rgba / 标准颜色名 背景颜色 
background-image  url(图片地址)  背景图片，支持本地/网络图片，不支持渐变 
background-repeat  no-repeat  /  repeat  /  repeat-x  /  repeat-y  背景图片重复规则 
background-size  cover  /  contain  /  stretch  背景图片缩放规则，同 image 组件 resize 属性 
background-position  left/center/right top/center/bottom  / px 单位 背景图片定位 
border  宽度 样式 颜色  边框简写，仅支持 solid 实线样式 
border-width / border-style / border-color 对应值 单方向边框属性 
border-left / border-right / border-top / border-bottom 对应值 单方向边框简写

2. scroller
 
组件用途
 
原生滚动容器，用于长内容的纵向/横向滚动，性能优先，适配 IoT 设备长列表场景
 
独有属性
 
属性名 类型 默认值 说明 
scroll-x Boolean false 开启横向滚动 
scroll-y Boolean true 开启纵向滚动 
show-scrollbar Boolean true 是否显示滚动条 
scroll-with-animation Boolean false 滚动时是否开启动画 
upper-threshold Number 50 距顶部/左边多远时（单位px），触发 scrolltoupper 事件 
lower-threshold Number 50 距底部/右边多远时（单位px），触发 scrolltolower 事件 
 
独有事件
 
事件名 回调参数 说明 
@scroll {scrollLeft, scrollTop, scrollWidth, scrollHeight} 滚动时持续触发，返回当前滚动位置 
@scrolltoupper 无 滚动到顶部/左边时触发 
@scrolltolower 无 滚动到底部/右边时触发 
@scrollend 无 滚动结束时触发 
 
独有插槽
 
默认插槽：滚动内容
 
强制约束
 
1. 必须设置固定 width 和 height，否则无法触发滚动效果
2. 内容宽高必须大于容器宽高，滚动才会生效
 
完整可运行示例
 
vue
  
<template>
  <scroller 
    scroll-y 
    show-scrollbar 
    scroll-with-animation
    style="width: 320px; height: 400px; border: 1px solid #eee;"
    @scroll="handleScroll"
    @scrolltolower="loadMore"
    @scrollend="handleScrollEnd"
  >
    <div v-for="i in 20" :key="i" style="height: 50px; line-height: 50px; border-bottom: 1px solid #f0f0f0; padding-left: 16px;">
      列表项 {{ i }}
    </div>
  </scroller>
</template>

<script>
export default {
  methods: {
    handleScroll(e) {
      console.log('当前滚动位置：', e.scrollTop);
    },
    loadMore() {
      console.log('触底加载更多');
    },
    handleScrollEnd() {
      console.log('滚动结束');
    }
  }
}
</script>

4. image
 
组件用途
 
图片渲染组件，原生解码，支持网络图、Base64、本地图片，内置 LRU 图片缓存机制
 
独有属性
 
属性名 类型 默认值 说明 
src String - 必选，图片地址，支持 URL/Base64/require 本地路径 
resize String stretch 图片缩放模式：stretch(拉伸填充)、cover(裁剪填充)、contain(完整显示) 
alt String '' 图片加载失败时的占位文字 
lazy-load Boolean false 是否开启懒加载，仅在 scroller 中生效 
 
独有事件
 
事件名 回调参数 说明 
@load {success: Boolean, size: {naturalWidth, naturalHeight}} 图片加载完成时触发，返回加载结果和原始尺寸 
@error 无 图片加载失败时触发 
 
独有插槽
 
无
 
强制约束
 
1. 必须设置固定 width 和 height，不支持百分比宽高
2. 本地图片必须使用  require()  方式引入
 
完整可运行示例
 
vue
  
<template>
  <image
    src="https://vuejs.org/images/logo.png"
    resize="contain"
    lazy-load
    style="width: 200px; height: 200px; border-radius: 8px;"
    @load="handleLoad"
    @error="handleError"
  ></image>
</template>

<script>
export default {
  methods: {
    handleLoad(e) {
      if (e.success) {
        console.log('图片加载成功，原始尺寸：', e.size);
      }
    },
    handleError() {
      console.log('图片加载失败');
    }
  }
}
</script>
 
 
 
 
5. slider
 
组件用途
 
原生滑动选择器，用于连续数值选择，适配 IoT 设备触摸滑动交互
 
独有属性
 
属性名 类型 默认值 说明 
min Number 0 最小值 
max Number 100 最大值 
step Number 1 步长，必须大于 0 
v-model Number 0 双向绑定当前选中值 
disabled Boolean false 是否禁用滑动 
active-color String #1890ff 滑块选中区域的颜色 
background-color String #ccc 滑块未选中区域的颜色 
 
独有事件
 
事件名 回调参数 说明 
@change 当前值 滑动结束/值改变时触发，返回最终值 
@moving 当前值 滑动过程中持续触发，返回实时值 
 
独有插槽
 
无
 
强制约束
 
必须设置固定宽度
 
完整可运行示例
 
vue
  
<template>
  <div style="padding: 20px;">
    <slider
      :min="0"
      :max="100"
      :step="5"
      v-model="sliderVal"
      active-color="#1890ff"
      :disabled="false"
      style="width: 300px; height: 40px;"
      @change="handleChange"
      @moving="handleMoving"
    ></slider>
    <text style="margin-top: 16px; display: block;">当前选中值：{{ sliderVal }}</text>
  </div>
</template>

<script>
export default {
  data() {
    return {
      sliderVal: 50
    }
  },
  methods: {
    handleMoving(val) {
      console.log('滑动中实时值：', val);
    },
    handleChange(val) {
      console.log('滑动最终值：', val);
    }
  }
}
</script>
 
 
 
 
6. canvas
 
组件用途
 
2D 画布组件，支持标准 Canvas 2D API，用于自定义图形绘制、特效渲染，支持 GPU 硬件加速
 
独有属性
 
无
 
独有事件
 
无
 
独有插槽
 
无
 
强制约束
 
1. 必须设置固定 width 和 height
2. 必须在  mounted  生命周期中获取绘图上下文，确保 DOM 渲染完成
 
完整可运行示例
 
vue
  
<template>
  <canvas ref="drawCanvas" style="width: 300px; height: 300px; background: #f5f5f5;"></canvas>
</template>

<script>
export default {
  mounted() {
    // 获取画布实例和绘图上下文
    const canvas = this.$refs.drawCanvas;
    const ctx = canvas.getContext('2d');

    // 绘制蓝色矩形
    ctx.fillStyle = '#1890ff';
    ctx.fillRect(50, 50, 200, 200);

    // 绘制白色圆形
    ctx.beginPath();
    ctx.arc(150, 150, 60, 0, 2 * Math.PI);
    ctx.fillStyle = '#fff';
    ctx.fill();

    // 绘制边框
    ctx.strokeStyle = '#333';
    ctx.lineWidth = 2;
    ctx.strokeRect(50, 50, 200, 200);
  }
}
</script>

第三部分：$falcon 全局单例对象完整规范
 
一、整体定义
 
 $falcon  是 HaaS UI 框架注入的顶层只读全局单例对象，Vue 组件内可通过  this.$falcon  访问，全局上下文可直接访问，无需额外引入。它是框架所有能力的统一入口，完整包含 6 大核心模块：环境信息、应用实例、系统 JSAPI、离线文档查询工具、全局事件总线、内置工具集。
 
 
 
二、核心模块 1：$falcon.env 环境信息
 
用途
 
只读属性集合，提供当前运行环境、设备、框架、屏幕的全量静态信息，不可修改
 
全量属性表
 
属性名 类型 示例值 说明 
platform String  AliOS Things  /  Linux  /  Windows  /  MacOS  当前运行的操作系统平台 
frameworkVersion String  2.3.0  HaaS UI 框架版本号

deviceModel String  树莓派4B  /  全志D1  硬件设备型号 
isDebug Boolean  true  /  false  当前是否为调试模式 
appId String  com.haas.demo  当前应用的唯一 ID

六、核心模块 5：$falcon.eventBus 全局事件总线
 
用途
 
全局订阅发布工具，用于跨组件、跨页面的通信，无层级限制，解耦业务逻辑
 
全量方法表
 
方法名 入参格式 说明 
on  eventName: String, callback: Function  订阅全局事件，事件触发时执行回调函数 
once  eventName: String, callback: Function  订阅单次全局事件，触发后自动销毁订阅 
emit  eventName: String, ...args  触发全局事件，可传递任意数量的参数给订阅回调 
off  eventName: String, callback?: Function  取消订阅；不传 callback 则取消该事件的所有订阅 
 
调用示例
 
vue
  
<!-- 页面A：订阅事件 -->
<script>
export default {
  mounted() {
    // 订阅设备状态变化事件
    $falcon.eventBus.on('deviceStatusChange', this.handleStatusChange);
    // 订阅单次事件
    $falcon.eventBus.once('appUpdate', this.handleUpdate);
  },
  beforeDestroy() {
    // 页面销毁时取消订阅，避免内存泄漏
    $falcon.eventBus.off('deviceStatusChange', this.handleStatusChange);
  },
  methods: {
    handleStatusChange(status) {
      console.log('设备状态更新：', status);
    },
    handleUpdate() {
      console.log('应用更新完成');
    }
  }
}
</script>

<!-- 页面B：触发事件 -->
<script>
export default {
  methods: {
    changeDeviceStatus() {
      // 触发全局事件，传递参数
      $falcon.eventBus.emit('deviceStatusChange', 'online');
      $falcon.eventBus.emit('appUpdate');
    }
  }
}
</script>

deepClone  target: Object/Array  Object/Array 深拷贝对象/数组，解决引用类型赋值问题

function _collectFalconEventIds(name, callback)
{
  const evtList = $falcon.eventMap[name]
  if (evtList) {
    if (callback) {
      const index = evtList.findIndex(item => item.callback === callback || item.id === callback);
      if (index !== -1) {
        return [evtList[index].id]
      }
    } else {
      return evtList.map((item) => item.id)
    }
  }
  return []
}

class PageRes extends $falcon.Page {
  constructor() {
    super()
    this.falconOnTokens = []  // [[token, name], ...]
    this.timeoutTokens = new Set()  // [token, ...]
    this.intervalTokens = new Set()  // [token, ...]
  }

  on(name, callback) {
    const token = $falcon.on(name, callback)
    this.falconOnTokens.push([token, name])
    return token
  }

  off(name, callback) {
    const falconOnTokens2 = []
    let idsWillRemoved = _collectFalconEventIds(name, callback)
    DEBUG && console.log(`idsWillRemoved ${JSON.stringify(idsWillRemoved)}`)
    idsWillRemoved = new Set(idsWillRemoved)
    for (let [token, name] of this.falconOnTokens) {
      if (idsWillRemoved.has(token)) {
        continue
      }
      falconOnTokens2.push([token, name])
    }
    this.falconOnTokens = falconOnTokens2

    $falcon.off(name, callback)
  }
  trigger(name, options) {
    $falcon.trigger(name, options)
  }
  setTimeout(func, ms) {
    const token = setTimeout(() => {
      this.timeoutTokens.delete(token)
      func()
    }, ms)
    this.timeoutTokens.add(token)
    return token
  }
  setInterval(func, ms) {
    const token = setInterval(func, ms)
    this.intervalTokens.add(token)
    return token
  }
  clearTimeout(token) {
    this.timeoutTokens.delete(token)
    clearTimeout(token)
  }
  clearInterval(token) {
    this.intervalTokens.delete(token)
    clearInterval(token)
  }
  release() {
    for (let [token, name] of this.falconOnTokens) {
      DEBUG && console.log(`release $falcon.on token ${token}`)
      $falcon.off(name, token)
    }
    this.falconOnTokens.length = 0
    for (let token of this.timeoutTokens) {
      DEBUG && console.log(`release setTimeout token ${token}`)
      clearTimeout(token)
    }
    this.timeoutTokens.clear()
    for (let token of this.intervalTokens) {
      DEBUG && console.log(`release setInterval token ${token}`)
      clearInterval(token)
    }
    this.intervalTokens.clear()
  }
}

function formatDateTimeStr(now)
{
  const year = now.getFullYear();
  const month = (now.getMonth() + 1).toString().padStart(2, '0');
  const day = now.getDate().toString().padStart(2, '0');
  const hour = now.getHours().toString().padStart(2, '0');
  const minute = now.getMinutes().toString().padStart(2, '0');
  const second = now.getSeconds().toString().padStart(2, '0');

  return `${year}-${month}-${day} ${hour}:${minute}:${second}`
}
